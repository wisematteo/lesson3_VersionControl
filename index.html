Version Control

For files that are constantly being changed or expanded on, a version control system can keep track of all versions of a file. For business who use the internet and keep files saved online, version control clients can help save files if they are lost or accidents happen. Using a distributed version control system such as github, for our class we will always have a backup for our files. Git can save snapshots of files and when a user commits, the files are saved to a project in git. This method makes github a very efficient version control system, especially for its capabilities.

The Three States

Committed, modified, and staged are the three states your files can be in within a version control system. Committing a file is saving it into a local repository. Modifying a file means that you have changed the file but have not saved it yet. Workflow within git follows a pattern in which a user is modifying files in a work-tree, then staging the files, which takes snapshots and adds them to the staging area. Finally committing a file saves it permanently to a git directory.

Staging Modified Files

If a change is made to a file that has been tracked it eventually should be saved. If nothing is done the file will appear as ‘changes not staged for commit’, if looked at using the git status command. To stage the file the next step is to use the git add command. Git add is a multipurpose command and can be used for new files, old files, and files that resolve merge-conflicting files. The file will be staged, if changes are made then there will be both staged and un-staged files. Git commit will send files to a repository and if changes are made then git add must be done to stage the files.

Git Commands

Git tags are the key to using the VCS, basic commands range from git checkout and checkout -b, git commit, and git branch. Vital tags to use for a local repository are git fetch, git pull, git push, and git tag. Undo tags include git reset and git reserve and to combine breaches, the tags are git merge and git rebase. Many of the tags have multiple functions. For example, git push can be used for staged, un-staged, and un-merged files. Git commit is also a very important command.

Undoing Tags

When mistakes are made it is important to have a way to go back or undo what you just did. Luckily, the git –amend option allows users to go back on commits that have files missing, an incorrect message, or any other issue. The command takes your last commit back to the staging area, you can edit this file but it will overwrite that last commit. Also, if a user has two differing un-staged files and accidentally commits both, first using the git status tag they can be checked, followed by the rare use of the git reset tag. The git reset tag can be very dangerous to use, especially if used with double dashes (--)
